Done

A simple to-do list app allowing each item to be marked as done or simply delete by swiping to left.

I developed this app to practice using Realm and CocoaPods. I have used Realm to allow users to create multiple lists/categories and save them locally. In order to replicate the swipe to delete behaviour, I have installed SwipeCellKit.

I have also used ChameleonFramework to assign random colours to each category that is persisted and creates a nice UI.

I have created the same app without any pods and only using Core Data for local data storage which can be accessed here.

MessageMe

Simple messaging app built in MVC design pattern using Firebase Authentication and Firestore.

I have developed this app to practice Firebase and TableViews further. TableView is the bread and butter of a good iOS Developer so I wanted to work on TableView as much as possible to improve my skills.

I have used Firebase Authentication to allow users to sign up/log in. The users can message each other and the messages are persisted online using Cloud Firestore.

I have used IQKeyboardManagerSwift to be able to easily allow the user to move up the TextField with the keyboard on tap across all orientations and devices.

One of my challenges in this project was checking if the user has logged in during app launch and to change the RootViewController accordingly, making sure NavigationController is initiated properly.

Although I can now see how basic the solution was to all my struggles, at the time I had to spent considerable time to fully understand how different components like SceneDelegate, NavigationController, ViewController work and interact with each other.

The knowledge I gained in this project has been really helpful in my other projects since then - and helped me to avoid many sleepless nights! 😅

Cloudy

Weather forecast app that allows the user to either look for weather at their current location based on the GPS data or search for another city manually.

I have used Core Location to acquire user location and provide weather forecast for their current location. This is done automatically when the app launches but can also be obtained by clicking the location button on the left side of the search bar.

The user can also search for any location using the search bar at the top.

The UI is updated according to the weather (similar to Apple's Weather app)

Built using Open Weather API and MVC design pattern.

I am currently working on a SwiftUI version of this app to practice using MVVM design pattern.

Bookie

A book manager app that allows users to search for books and add them to different lists (Currently Reading, Read, Want to Read). I have used Google Books API to be able to perform searches and retrieve information.

The data is saved online using Firebase and persisted locally to allow offline access.

I have used Firebase Authentication to allow users to sign up/log in. The books saved to the lists are persisted online using Cloud Firestore with offline cache.

I have also used a simple Core Data model to be able to update UI quickly.
